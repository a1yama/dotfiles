## これは何か

`claude-diff` CLI を使い、編集差分をブラウザの差分ビューアで表示する。ユーザーは行単位でコメントを残し、Claude はそのコメントをバックグラウンド監視で自動的に拾って対応する。IDE を開く必要も、Claude を中断する必要もない。

ビューアと監視はリポジトリを汚さない（フィードバックは `~/.local/state/claude-diff/<repo-hash>/` に保存）。

## 使い方

### 1. ビューアを起動する（バックグラウンド）

`run_in_background: true` で起動し、ブラウザを開く:

```bash
claude-diff serve
```

ポート競合時は `claude-diff serve --port 4600` のように変更する。デフォルトは 4517。

起動後、ユーザーに「ブラウザ (http://127.0.0.1:4517/) で差分を確認し、気になる行の `+` からコメントを残してください」と伝える。

- ブラウザ上部のセレクタで **未コミット差分 / ブランチ全体** を切替できる。任意のコミット範囲（例 `HEAD~3`, `a..b`）はテキスト欄に入力。
- 各行ホバーで出る `+` を押すとコメント入力欄が開く。

### 2. フィードバックを自動で拾う（重要）

サーバ起動後、`run_in_background: true` で監視コマンドを起動する:

```bash
claude-diff watch
```

これはフィードバックファイルに新規コメントが入るまでブロックする。ユーザーがブラウザでコメントを送信するとコマンドが終了し、ハーネスが Claude を再起動する。**待機中はトークンを消費しない。**

監視が新しいフィードバックを返したら:

1. コメント内容（`[id] file:line` 形式）に従って修正する。
2. 対応したコメントを `claude-diff resolve <id> [<id>...]` で resolved にする（ブラウザ上のバッジが緑になる）。
3. 再び `claude-diff watch` をバックグラウンド起動して次のコメントを待つ。

`TIMEOUT` が返った場合（既定30分）は、まだレビュー中か確認するためユーザーに一声かけるか、監視を再開する。

### 3. 終了

レビューが終わったら、起動したバックグラウンドのサーバ/監視プロセスを停止する。

## 軽量モード（tmux）

ブラウザを使わず、別 tmux ペインに色付き差分を定期更新表示するだけでよい場合:

```bash
claude-diff tmux              # 未コミット差分を別ペインに表示
claude-diff tmux --range branch
```

`delta` が入っていれば自動で使う（なければ `less -R`）。この場合フィードバックはブラウザ経由ではないため、ユーザーは口頭/プロンプトで指示する。

## サブコマンド一覧

| コマンド | 説明 |
|---|---|
| `claude-diff serve [--port N] [--no-open]` | 差分ビューアを起動 |
| `claude-diff watch [--timeout S] [--interval S]` | 新規コメントが入るまでブロック |
| `claude-diff resolve <id...>` | コメントを resolved に（`all` で全件） |
| `claude-diff status` | pending/delivered/resolved の件数と一覧 |
| `claude-diff tmux [--range R]` | tmux ペインに色付き差分を定期表示 |
