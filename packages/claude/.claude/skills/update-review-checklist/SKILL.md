---
description: コードレビューで見逃したミスや新たに気づいた落とし穴をチェックリストに追加する
---

引数が指定されていない場合は、以下の情報をユーザーに質問してください：

1. **言語**: go / typescript / javascript / php / python / rust のいずれか
2. **カテゴリ**: よくあるミスのカテゴリ（例: エラーハンドリング、並行処理、型安全性など）
3. **ミスの概要**: 一行で簡潔に
4. **詳細説明**: ミスの内容や問題点
5. **コード例**（オプション）: 該当する場合は NG/OK の例

引数が指定されている場合は、以下の形式で解釈してください：
```
/update-review-checklist <言語> <カテゴリ> "<ミスの概要>" "<詳細説明>" [コード例]
```

例:
```
/update-review-checklist go "定数定義" "iotaの途中に定数追加で値がずれる" "iota で定義された定数の途中に新しい定数を追加すると、それ以降の定数の値が全てずれてしまう"
```

## 実行手順

1. 対象言語のチェックリストファイルを読み込む
   - ファイルパス: `~/.claude/skills/code-review/languages/<言語>.md`

2. 該当するカテゴリが存在するか確認
   - 存在する場合: そのカテゴリ配下に新しいミスを追加
   - 存在しない場合: 新しいカテゴリセクションを作成して追加

3. 以下の形式でエントリを追加:
   ```markdown
   - **<ミスの概要>**
     - 説明: <詳細説明>
     - 例: （コード例がある場合）
       ```<言語>
       <コード例>
       ```
   ```

4. ファイルを更新後、ユーザーに完了を報告

## 注意事項

- 既に同じ内容のチェック項目が存在しないか確認してから追加する
- カテゴリ名は既存のカテゴリ名と統一する（例: 「エラーハンドリング」「並行処理」「型安全性」など）
- 簡潔で分かりやすい表現を心がける
